OmegateDictate: Speech-to-Text Plugin

The Speech-to-Text plugin allows you to easily integrate speech recognition functionality into your web application. It utilizes the Web Speech API to convert spoken language into written text.

  • Version: 1.0
  • Author: Issa Sukatu Abdullahi
  • Created: 21 June, 2023
  • Last Update: 22 June, 2023

Getting Started

To use the Speech-to-Text plugin, follow these steps:

Include the necessary CSS and JavaScript dependencies in your HTML file. Add the following lines within the tag: 
<!DOCTYPE html>
<html lang="en">
<head>


  <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/twitter-bootstrap/5.3.0/css/bootstrap.min.css" />
  <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/5.15.4/css/all.min.css" />
  <script src="path/to/speech-to-text-plugin.js"></script>


</head>
</body>
</html>

Replace "path/to/speech-to-text-plugin.js" with the actual path to the JavaScript file containing the plugin code.

Create the HTML structure for the speech recognition interface. You can use the following example as a starting point:

Include the necessary CSS and JavaScript dependencies in your HTML file. Add the following lines within the tag: 
<!DOCTYPE html>
<html lang="en">
<head>

   
      <div class="container mt-5">
        <h1>Speech to Text</h1>
        <div class="my-3">
          <button id="start-btn" class="btn btn-primary"> <i class="fa fa-microphone"></i> Start</button>
          <button id="pause-btn" class="btn btn-secondary" disabled> <i class="fa fa-pause"></i> Pause</button>
          <button id="continue-btn" class="btn btn-success"> <i class="fa fa-play"></i> Continue</button>
          <button id="end-btn" class="btn btn-danger" disabled> <i class="fa fa-stop"></i> End</button>
        </div>
        <div class="mb-3">
          <textarea id="output" class="form-control" rows="5"></textarea>
        </div>
      </div>
    

</head>
</body>
</html>

Customize the layout and styling as needed.

Initialize the plugin by calling the OmegaDictate() function. Place the following line of code at the end of your HTML file, just before the closing tag:

  <html lang="en">
  <head>

  ============================== -->

  <body>

    OmegaDictate();
    </script>
  
  </body>
  </html>

Plugin Functionality

The Speech-to-Text plugin provides the following functionality:

  • Start Button: Clicking the "Start" button initiates the speech recognition process. If microphone permission hasn't been granted, it will request permission from the user. Once started, the button becomes disabled, and other relevant buttons are enabled.
  • Pause Button: Clicking the "Pause" button pauses the speech recognition. The button becomes disabled, and the "Continue" and "End" buttons become enabled.
  • Continue Button: Clicking the "Continue" button resumes the speech recognition after it has been paused. The button becomes disabled, and the "Pause" and "End" buttons become enabled.
  • End Button: Clicking the "End" button stops the speech recognition process. The button becomes disabled, and other buttons are enabled. Note that stopping the process also clears the recognition output.
  • Output Textarea: The textarea with the ID "output" displays the recognized text. It updates in real-time as speech is transcribed. You can also manually input text into the textarea, which will be considered as part of the final transcript.

Plugin Events

The Speech-to-Text plugin triggers the following events:

  • start() - Blue Color
  • pause() - Fired when the speech recognition is paused.
  • continue() - Fired when the speech recognition resumes after being paused.
  • end() - Fired when the speech recognition process ends.
  • error() - Fired if an error occurs during the speech recognition process.

You can listen to these events and add custom logic as needed.

Browser Compatibility

The Speech-to-Text plugin relies on the Web Speech API, which is supported in modern browsers, including Google Chrome and Mozilla Firefox. Please refer to the Web Speech API documentation for the latest browser compatibility information.

Limitations

The Logo Container can be found in the Header Container - <header>

  • The accuracy of speech recognition may vary depending on factors such as ambient noise and pronunciation.
  • The plugin requires user permission to access the microphone. If permission is denied, the speech recognition functionality will not work.
  • The plugin is currently designed for the English language (recognition.lang = "en-US"). Modifying it to support other languages may require additional customization.

Troubleshooting

If you encounter any issues with the Speech-to-Text plugin, consider the following troubleshooting steps:

  1. Check browser compatibility: Ensure that you are using a browser that supports the Web Speech API.
  2. Verify microphone access: Make sure that the browser has permission to access the microphone. You can check the browser settings to grant or revoke access.
  3. Review the console: If there are any errors, check the browser console for error messages. The plugin may log relevant information or error details.
  4. Seek community support: If the issue persists, reach out to the developer community or consult relevant documentation to get assistance.

Conclusion

The Speech-to-Text plugin simplifies the integration of speech recognition functionality into your web application. By following the steps outlined in this documentation, you can quickly enable users to convert spoken language into written text. Customize the plugin according to your specific requirements and enhance the user experience with voice input.
